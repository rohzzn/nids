Slide 1: Abstract

  • Wireless networks are increasingly vulnerable to attacks.

  • IDSs are key for anomaly detection and attack prevention.

  • Traditional data mining methods have high false alarm rates and poor generalization.

  • We propose AS-CNN, combining ADASYN and an improved CNN (SPC-CNN).

  • ADASYN oversamples minority classes to balance sample distribution.

  • SPC-CNN uses channel splitting, hierarchical processing, and soft attention fusion for diverse feature extraction.

  • AS-CNN simulations on NSL-KDD dataset show:

    • 4.60% higher accuracy than traditional models.

    • 11.34% increased detection rate.

    • 15.58% reduced false alarm rate

Slide 2: Introduction

  • Wireless networks are being widely adopted, but their open nature makes them vulnerable to attacks

  • Network security technologies like firewalls and encryption are not foolproof, so attack identification mechanisms (IDS) are crucial

  • Different types of network attacks include user to root (U2R), denial of service (DoS), remote to local (R2L), and probe attacks

  • IDS aims to classify these attacks correctly based on learned characteristics to detect anomalies early

  • The diversity and evolution of network attacks pose severe challenges to traditional IDSs

Slide 3: Existing Systems

Traditional Machine Learning approaches:

  • Techniques like Naive Bayes, Decision Trees, Support Vector Machines have been applied to IDS

  • Require extensive feature engineering and struggle to adapt to evolving network environments

Deep Learning approaches:

  • Deep neural networks can automatically extract features from raw network traffic data

  • Models like Deep Belief Networks (DBNs), Recurrent Neural Networks (RNNs), and Convolutional Neural Networks (CNNs) used

  • CNNs particularly successful due to local perception and weight sharing capabilities

Slide 4: Limitations

Imbalanced data distribution:

  • Real-world network traffic datasets often have imbalanced class distributions

  • With majority of normal traffic and fewer attack samples

  • Can lead to models being biased towards frequent classes and ignoring minority attack classes

Inter-channel redundancy in CNNs:

  • Output feature maps of convolutional layers may contain redundant information across channels

  • Existing CNN-based IDS methods do not effectively address this redundancy, impacting performance

  1. High false alarm rate - Generates excessive false positives, leading to alert fatigue.

  2. Low recognition accuracy - Struggles to accurately differentiate between normal and malicious activity.

  3. Poor generalization ability - Limited adaptability to evolving threats and network environments.

 

Slide 5: System Architecture

Overview of the proposed AS-CNN model's architecture and workflow:

Four main parts: data preprocessing, network design, model training/testing, and evaluation.

  • Data preprocessing involves numerical encoding, normalization, and ADASYN oversampling.

  • Network design presents the proposed SPC-CNN with channel splitting, hierarchical processing, and attention fusion.

  • Model training adjusts parameters, testing evaluates performance after each cycle.

  • Evaluation calculates standard metrics like accuracy, detection rate, and false alarm rate.

  • ADASYN balances class distribution in data preprocessing stage.

  • SPC-CNN architecture aims to address limitations of traditional CNNs.

  • Systematic process to validate performance improvements of AS-CNN model.

Slide 6: Proposed System - Modules (ADASYN)

  • Mitigating Sensitivity to Sample Size - Ensures smaller samples are not overlooked during training.

  • Balance the sample distribution - Generates synthetic samples for minority classes.

Slide 7: Proposed System - Improved CNN (SPC-CNN Architecture)

  • Integration of Split Convolution Module (SPC) - Enhances feature diversity and eliminates interchannel information redundancy during training.

  • Increasing Diversity of Features - Captures more nuanced patterns in network traffic data.

  • Eliminating Redundancy in Model Training - Reduces the impact of interchannel information redundancy, Improves efficiency and effectiveness of model training process.

Slide 8: Proposed System - Integration of ADASYN and SPC-CNN

  • ADASYN preprocesses data, balancing distribution; SPC-CNN extracts features from oversampled data.

  • Synergy: ADASYN prevents bias to majority, SPC-CNN enhances representation capability.

Slide 9: Results and Analysis - Simulation Setup

  • The proposed AS-CNN method was evaluated on the widely-used NSL-KDD dataset

    • The dataset consists of three subsets: KDDTrain, KDDTest+, and KDDTest-21

    • KDDTest+ and KDDTest-21 contain different attack types to test generalization ability

  • The evaluation metrics used were:

    • Accuracy (ACC): Overall classification accuracy

    • Detection Rate (DR): Ability to correctly identify attack samples

    • False Alarm Rate (FAR): Rate of misclassifying normal samples as attacks

  • TP = True Positives

  • TN = True Negatives

  • FP = False Positives

  • FN = False Negatives
